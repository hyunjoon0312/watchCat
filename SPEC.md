## 1. 프로젝트 개요
---
### 카테고리
> 생산성 / 업무

### 한 줄 정의
맥 앱 사용시간 자동 기록기 (상태바 마스코트로 기록 상태를 표시)

### 배경
사용자는 하루 동안 여러 앱과 웹을 오가며 일하지만, 실제로 어디에 시간을 쓰는지 체감과 기록이 다르다. 수동 기록은 번거롭고 누락되기 쉬워 자동 기록 도구가 필요하다.

### 사용자 문제
하루 동안 앱/웹 사용시간을 정확히 알기 어렵고, 수동 기록은 번거롭다. 또한 잠금/슬립/자리비움까지 기록되면 데이터가 왜곡된다.

### 해결 방식
백그라운드에서 활성 앱을 자동 추적해 사용시간을 누적하고, 크롬은 활성 탭의 웹페이지(도메인/URL)를 함께 기록한다. 상태바에서 '지켜보는 고양이(watchCat)' 마스코트가 기록 상태를 직관적으로 표시한다. 화면 잠금/슬립/사용자 입력 유휴 시 기록을 자동 중지한다. 모든 데이터는 로컬에만 저장되며 외부로 전송되지 않는다.

### 차별점
- 상태바 마스코트(고개 돌리는 고양이) 애니메이션으로 ‘기록 중’ 상태를 즉시 인지
- 크롬 웹페이지 단위 기록 + 잠금/슬립/유휴 제외로 데이터 신뢰도 확보
- 텔레메트리·외부 전송 없는 완전 로컬 동작

### 타겟 사용자
맥을 업무/학습에 사용하는 개인 사용자(개발자, 디자이너, 학생, 프리랜서 등)로, 생산성 분석이나 자기관리 목적의 사용시간 기록이 필요한 사람.

### 사용 시나리오
사용자가 watchCat을 실행(혹은 로그인 시 자동 실행)하면 상태바에 고양이가 나타나 주변을 두리번거리며 ‘지켜본다’. 사용자가 VS Code, Chrome 등으로 작업하는 동안 앱이 자동으로 활성 앱/웹페이지별 시간을 누적한다. 자리를 비워 잠금/슬립이 되거나 5분간 입력이 없으면 고양이가 눈을 감고 Zzz를 띄우며 기록을 멈춘다. 복귀하면 자동으로 재개된다. 하루가 끝나면 상태바 메뉴에서 오늘 요약을 본다.

### 제품 목표
사용자가 맥에서 하루 동안 어떤 앱(및 웹사이트)을 얼마나 사용했는지 자동으로 기록하고, 상태바에서 기록 중임을 직관적으로 보여준다.

### 핵심 KPI
- **상태 전환 지연**: 이벤트 수신(t0)부터 상태 반영(t1)까지 `t1 − t0 ≤ 1s` 달성률
- **비활성 누적 방지**: 잠금·슬립·유휴 구간이 어떤 세션 누적에도 합산되지 않음 (합계 검증)
- **카테고리 미분류 비율**: 사용자가 수동 분류하지 않은 앱이 차지하는 시간 비중 (자동분류는 v0.2)
- **로컬 활성 사용일수**: 앱이 1회 이상 세션을 기록한 일자 수 (앱 자체 통계, 외부 미전송)

### 리스크/이슈
macOS 권한(접근성/화면 기록/Apple Events) 요구로 설치/온보딩이 복잡해질 수 있음. 크롬 탭 정보 접근 제한 또는 API 변경 리스크. 개인 정보(방문 URL) 기록에 대한 민감도. 배터리/성능 영향.

### 비기능 요건
- **저장**: SQLite 로컬 DB, 위치 `~/Library/Application Support/watchCat/`. 보존 기간 기본 90일(설정 가능). 외부 전송 없음.
- **성능**: 유휴 시 CPU < 1%, 메모리(RSS) < 50 MB.
- **개인정보**: 평문 로컬 저장. URL 쿼리/페이지 제목 등 민감 필드는 로그에 남기지 않음. 사용자는 일자별/전체 데이터 삭제 권리 보유(v0.2 기능으로 제공).
- **접근성**: 시스템 '동작 줄이기' 설정 ON 시 마스코트 애니메이션을 정적 아이콘으로 폴백.

### 기술 스택 / 환경
- 최소 OS: macOS 14 Sonoma
- 언어/UI: Swift / SwiftUI
- DB: SQLite (GRDB)
- 텔레메트리: 없음

### 사용자 역할
> 사용자

### 디바이스 (기기)
> macOS 데스크톱/노트북

### 시간 처리 규칙(공통)
- **타임존**: 사용자 시스템의 로컬 타임존 고정. 시스템 타임존 변경 시 과거 데이터는 재해석하지 않음.
- **자정 경계**: 자정을 걸치는 세션은 **시작 일자**에 100% 귀속한다 (분할하지 않음).
- **다중 모니터/Spaces**: macOS가 보고하는 최전면 활성 앱을 단일 진실로 사용.

## 0. 최초 실행 및 권한 온보딩
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

앱이 정상 동작하려면 macOS 권한이 필요하다. 첫 실행 시 명확한 순서로 권한을 요청하고, 거부 시에도 부분 동작 가능한 모드로 진입한다. 자동 시작도 기본 ON으로 설정한다.

**수용 기준:**
```
1. ○ 첫 실행 시 권한 안내 화면이 표시되며, 사용자가 다음 단계로 진행하면 시스템 권한 시트가 차례로 뜬다.
2. ○ 권한이 모두 거부되어도 앱은 종료되지 않고 '부분 동작 모드'로 진입(상태바 표시는 유지, 기록 기능은 비활성).
3. ○ 설정에서 권한 상태를 확인하고, 부족한 권한의 시스템 환경설정 위치로 직접 이동할 수 있다.
4. ○ 자동 시작이 기본 ON으로 설정되며, 설정에서 OFF 토글 가능.
```


### 0.1 권한 요청 순서
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

권한을 순서대로 요청한다: 접근성(Accessibility) → 화면 기록(Screen Recording) → 자동화(Apple Events for Chrome). 각 단계에서 권한이 필요한 이유를 한 문장으로 안내한다.


#### 0.1.1 권한별 사유 표시 및 요청

각 권한 요청 직전 화면에서 “이 권한은 …를 위해 필요합니다” 형식의 사유를 한 줄로 표시하고, 시스템 권한 시트를 띄운다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 0.2 권한 거부 시 부분 동작 모드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

사용자가 권한을 거부해도 앱이 종료되지 않고, 가능한 기능만 활성화된 상태로 유지된다. 상태바에 ‘권한 필요’ 상태를 표시하고, 설정 진입점을 제공한다.


#### 0.2.1 부족한 권한 안내 및 재요청 진입점

설정 화면에 권한 항목별 상태(✓ 허용 / ✗ 거부)와 ‘시스템 환경설정 열기’ 버튼을 제공한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 0.3 자동 시작 (Launch at Login)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

기본값은 ON. 설정에서 토글로 변경 가능. 토글 변경은 즉시 시스템에 반영된다.


### 0.4 첫 실행 안내 화면
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

watchCat의 동작 방식(자동 기록, 상태바 표시, 잠금/슬립/유휴 제외, 로컬 저장)을 2–3 화면으로 안내한다. 다시 보지 않기 옵션 포함.

## 1. 상태바 기록 상태 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

앱 실행 시 macOS 상태바에 watchCat 마스코트(고양이)를 표시해 ‘기록 중’임을 사용자가 즉시 알 수 있어야 한다.

**수용 기준:**
```
1. ○ 앱 실행 후 3초 이내에 상태바에 watchCat 마스코트가 나타난다.
2. ○ 기록 중일 때는 마스코트가 고개를 좌·우로 천천히 돌리는 애니메이션을 보인다.
3. ○ 기록이 일시중지(잠금/슬립/유휴)되면 마스코트가 눈을 감고 Zzz를 띄운다.
4. ○ 상태바 아이콘 클릭 시 메뉴(오늘 요약·일시중지 토글·설정·종료)가 열린다.
5. ○ 시스템 '동작 줄이기' 설정이 ON이면 애니메이션 없이 정적 아이콘으로 동작한다.
```


### 1.1 상태바 마스코트 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

watchCat 마스코트는 22×22pt 컬러 SVG 자산(흰 본체 + 검정 윤곽 + 핑크 코·귀안)이며, 다크/라이트 모드 모두에서 동일하게 표시된다(템플릿 이미지 아님). 자산 소스는 [assets/mascot/](assets/mascot/).


#### 1.1.1 상태바 표시 정책

다크/라이트 모드 모두에서 흰색 본체가 검정 윤곽으로 둘러싸여 명확히 식별된다. 상태바 공간이 부족할 때도 최소 표시(아이콘만)는 유지된다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 1.1.2 다양한 DPI/해상도 대응

레티나 및 외부 디스플레이의 다양한 DPI에서 픽셀 스냅 렌더링으로 선명도를 유지한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 1.2 기록 상태 애니메이션
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

‘기록 중’과 ‘일시중지’ 두 상태를 마스코트 애니메이션으로 명확히 구분한다.


#### 1.2.1 기록 상태 머신

상태는 `기록 중`과 `일시중지` 두 가지. 일시중지의 트리거는 (a) 화면 잠금, (b) 슬립, (c) 입력 유휴, (d) 사용자가 메뉴에서 수동 일시중지 — 네 가지이며, 어떤 트리거든 상태는 동일하게 ‘일시중지’.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 1.2.2 기록 중 — 두리번거리는 애니메이션

5프레임(`record-1-left` → `record-2-front` → `record-3-right` → `record-4-front` → `record-5-blink`) / 1.5초 주기. 프레임 1·3은 시선이 좌·우로 살피는 자세, 프레임 2는 약간 아래를 보는 정면, 프레임 4는 눈을 더 크게 뜨고 살짝 미소를 띤 정면(변화감), 프레임 5는 눈을 감는 깜빡임. 상태바가 가려지거나 사용자가 다른 Space에 있을 때는 애니메이션을 멈춰 배터리 절약.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 1.2.3 일시중지 — 눈 감김 + Zzz

3단계 단일 자산(`pause-1` → `pause-2` → `pause-3`)을 0.8초 간격으로 순환 표시. 각 자산은 눈을 감은 고양이 얼굴과 우상단 Zzz를 함께 그린 합성 스프라이트로, 단계가 올라갈수록 Zzz 글자가 하나씩 추가되어 점점 위로 올라간다. 일시중지 트리거 종류(잠금/슬립/유휴/수동)는 메뉴 텍스트에서 구분 표시.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 1.2.4 동작 줄이기 폴백

시스템 ‘동작 줄이기(Reduce Motion)’ 설정이 ON이면 애니메이션 없이 정적 자산을 사용한다. 기록 중은 `record-2-front`, 일시중지는 `pause-2` 자산을 고정 표시한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 1.3 앱 실행/종료 시 상태바 동작
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 1.3.1 단일 인스턴스 보장

이미 실행 중일 때 두 번째 인스턴스가 시작되면, 두 번째는 첫 인스턴스에 포커스를 넘기고 즉시 종료한다. 상태바 아이템은 항상 1개만 존재.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 1.3.2 종료 시 리소스 정리

앱 종료 시 진행 중인 세션을 종료 저장하고, 이벤트 구독·타이머를 해제한 뒤 상태바 아이템을 제거한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 1.4 상태바 메뉴 (오늘 요약 / 일시중지 / 설정 / 종료)

## 2. 활성 애플리케이션 사용시간 자동 기록
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

포그라운드 활성 애플리케이션을 감지하고, 앱별 사용시간을 자동으로 누적 기록한다.

**수용 기준:**
```
1. ○ 활성 앱이 바뀌면 이벤트 수신(t0)부터 누적 대상 변경(t1)까지 t1 − t0 ≤ 1s.
2. ○ 하루(로컬 일자) 기준으로 앱별 누적 사용시간을 상태바 메뉴 또는 오늘 요약 화면에서 확인할 수 있다.
3. ○ 기록은 사용자 조작 없이 자동으로 시작/유지된다.
4. ○ 자정을 걸치는 세션은 시작 일자에 100% 귀속한다.
```

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 2.1.2 전환 디바운스

짧은 포커스 이동(예: 알림 팝업)으로 누적 단위가 흔들리지 않도록 250ms 디바운스를 적용한다. 디바운스 시간은 상태 전환 지연 KPI(≤ 1s)를 위반하지 않는다.

세션 종료 트리거: (a) 활성 앱 변경, (b) 일시중지 진입(잠금/슬립/유휴/수동), (c) 앱 종료. 재개 시 새 세션을 시작한다. 각 세션은 시작 시각, 종료 시각, 앱 번들 ID를 보관.

## 3. 크롬 웹페이지(탭) 사용시간 기록
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

Chrome이 활성 앱일 때 활성 탭 정보를 식별해 웹페이지별 사용시간을 누적 기록한다. **MVP 범위는 Chrome만**이며, Safari/Arc/Firefox 등 다른 브라우저는 일반 앱으로 집계된다.

**수용 기준:**
```
1. ○ Chrome 활성 상태에서 탭(또는 페이지)이 바뀌면 t1 − t0 ≤ 1s로 전환을 감지한다.
2. ○ 하루 기준으로 웹페이지(기록 단위에 따라 도메인/URL/제목)별 누적 사용시간을 확인할 수 있다.
3. ○ 웹페이지 기록 단위 기본값은 ‘도메인’이며, URL·제목은 옵트인이다.
4. ○ Chrome 사용시간 = 해당 일자의 모든 웹페이지 시간 합과 일치한다.
```


### 3.1 크롬 활성 감지 및 탭 정보 수집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 3.1.1 크롬 활성 여부 판별

활성 앱이 Chrome일 때만 탭 정보를 수집한다. 비활성 시에는 마지막 탭을 고정하지 않고 ‘미수집’ 상태로 전환한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 3.1.2 수집 필드 정의

수집 필드는 도메인 / URL / 페이지 제목. 기본 저장은 도메인만이며, URL·제목은 설정에서 옵트인.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 3.2 탭 전환 감지 및 웹페이지별 누적
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 3.2.1 탭 전환 시 세션 분리

탭 변경을 감지하면 이전 웹페이지 세션을 종료하고 새 세션을 시작한다. 전환 감지 지연 ≤ 1s를 만족한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 3.2.2 앱-웹 합 일치 보장

Chrome의 앱 누적 시간 = 같은 일자의 모든 웹페이지 누적 시간의 합 (오차 없음).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 3.3 웹페이지 기록 단위 설정 (도메인 기본)
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 3.3.1 기록 단위 UI

설정에서 단위를 선택: **도메인(기본)** / URL / 제목. 단위 변경은 변경 시점 이후 데이터에만 적용되며, 과거 데이터는 변환하지 않는다.


잠금 이벤트 수신 시 진행 중인 세션을 종료하고 ‘일시중지(잠금)’ 상태로 전환. 해제 이벤트 수신 시 ‘기록 중’으로 복귀하고 새 세션을 시작.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 4.2 슬립 / 복귀 감지
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 4.2.1 슬립/복귀 이벤트 처리

슬립 진입 직전 세션을 종료 저장하고 ‘일시중지(슬립)’ 상태로 전환. 복귀 시 ‘기록 중’으로 복귀.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 4.3 일시중지 구간 누적 방지
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 4.3.1 타임라인 제외

일시중지 시작~종료 사이 시간은 어떤 세션에도 더하지 않는다. 합성 타임라인으로 합계 = 활성 합계 + 일시중지 합계인지 검증한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 4.3.2 자동 재개 안전장치

이벤트 누락 대비 60초 주기로 잠금/슬립/유휴 상태를 능동 점검해 상태를 보정한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


### 4.4 사용자 입력 유휴 감지
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

키보드/마우스 입력이 임계값 동안 없으면 ‘유휴(자리비움)’로 간주해 기록을 일시중지한다.


#### 4.4.1 유휴 임계값

기본 **5분**. 설정에서 1~30분 사이로 변경 가능. 변경은 즉시 적용된다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북


#### 4.4.2 유휴 진입 / 복귀

입력 없는 시간이 임계값을 초과하면 진행 세션을 종료하고 ‘일시중지(유휴)’로 전환. 다음 입력 발생 시 즉시 ‘기록 중’으로 복귀하고 새 세션 시작.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

## 5. 앱 카테고리화 (MVP — 수동만)
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전

사용자가 앱을 카테고리로 분류해 요약 화면에서 카테고리별 합계를 볼 수 있다. **MVP 범위는 수동 분류만**이며, 자동 추천 룰은 v0.2 로드맵.

**수용 기준:**
```
1. ○ 시드 카테고리(생산성·커뮤니케이션·오락·기타)가 기본 제공된다.
2. ○ 앱별로 카테고리를 지정·변경할 수 있다.
3. ○ 카테고리를 지정한 앱은 오늘 요약 화면에서 카테고리별 합계에 반영된다.
4. ○ 카테고리를 지정하지 않은 앱은 ‘미분류’로 집계되며, KPI ‘카테고리 미분류 비율’에 반영된다.
```


### 5.1 시드 카테고리
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

생산성 / 커뮤니케이션 / 오락 / 기타 — 4개를 기본 제공한다. 사용자 정의 카테고리 추가/편집은 v0.2.


### 5.2 카테고리 수동 편집
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: macOS 데스크톱/노트북

오늘 요약 화면의 앱 행에서 카테고리 지정 컨트롤을 통해 카테고리를 설정·변경할 수 있다. 변경은 과거 기록까지 소급 적용된다(앱 단위로 매핑 저장).

## 6. 검증 방법

명세의 수용기준을 어떻게 검증할지 정의한다. 구체 테스트 코드는 구현 단계에서 작성한다.

| 수용기준 | 검증 방법 |
|---|---|
| 상태 전환 지연 ≤ 1s | 모의 이벤트(앱 활성화/잠금/슬립/유휴)를 주입한 뒤 상태 머신 로그의 t0, t1 타임스탬프를 비교 |
| 비활성 합산 0 | 합성 타임라인(잠금/슬립/유휴 구간 포함)을 입력해 집계 후, 활성 합계 + 일시중지 합계 = 전체 경과시간인지 검증 |
| 단일 인스턴스 | 앱을 2회 연속 실행 → 두 번째 프로세스가 즉시 종료되고 상태바 아이템이 1개만 존재하는지 확인 |
| 자정 경계 시작일자 귀속 | 23:59 시작 → 00:30 종료 합성 세션이 시작 일자(전날)에 100% 합산되는지 검증 |
| 자동 재개 ≤ 1s | 잠금 → 해제 / 슬립 → 복귀 / 입력 유휴 → 입력 재개 시 마스코트 애니메이션 재생 시점 로그로 확인 |
| 데이터 보존 90일 | 91일 전 데이터가 자동 정리되는지, 보존 설정 변경이 즉시 반영되는지 확인 |
| 앱-웹 합 일치 | 같은 일자의 Chrome 앱 누적 시간과 모든 웹페이지 누적 시간의 합이 일치하는지 검증 |
| 인코그니토 미기록 | 인코그니토 탭만 사용한 합성 시나리오에서 도메인 시간이 0이고 ‘인코그니토(미상세)’ 시간이 일치하는지 확인 |

## 8. 구현 메모 (참고)

명세 본문이 아닌 참고용. 실제 API/라이브러리 선택은 구현 단계에서 확정한다.

- 활성 앱 감지: `NSWorkspace.didActivateApplicationNotification`
- 잠금/해제: `DistributedNotificationCenter` (`com.apple.screenIsLocked` / `com.apple.screenIsUnlocked`)
- 슬립/복귀: `NSWorkspace.willSleepNotification` / `didWakeNotification`
- 입력 유휴: `CGEventSource.secondsSinceLastEventType`
- 크롬 탭: Scripting Bridge / Apple Events
- DB: GRDB on SQLite
- 자동 시작: `SMAppService.mainApp` (macOS 13+)
- 상태바 마스코트 자산: 22×22pt SVG (viewBox 0 0 32 32), 컬러 자산이므로 `NSImage.isTemplate = false`. 흰 본체 + 검정 윤곽이라 다크/라이트 양쪽에서 동일 표시. 파일 목록: `record-{1-left,2-front,3-right,4-front,5-blink}.svg` (5), `pause-{1,2,3}.svg` (3). 자산 디렉토리: [assets/mascot/](assets/mascot/).
- 애니메이션은 상태바 가시 상태에서만 구동 (배터리)
- ‘동작 줄이기’: `NSWorkspace.shared.accessibilityDisplayShouldReduceMotion`
